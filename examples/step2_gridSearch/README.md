# Step 2 example

This folder documents one representative **rigid-body grid-search case**.

## Why there is not a fully hard-coded production example here

The Step 2 input depends on many geometric variables:

- simulation-cell bounds,
- grain-region bounds,
- orientation matrices for the two grains,
- rigid-body translation values,
- cutoff used for overlap removal,
- reference bulk energy for GB-energy calculation.

In the original workflow, these are naturally generated by Python rather than typed manually.

## What this example folder provides

- a named sample case,
- the key variables that must be defined,
- guidance on which values are case-specific and should come from your generator.

## Representative case label

`Sigma3_112 / Ni90Cr / Bonny-like grid-search case`

## Key variables to define before using the template

- `latticeConst`
- `xp0`, `xp1`, `yp00`, `yp55`, `zp0`, `zp1`
- `yp0`, `yp1`, `yp2`, `yp3`, `yp4`, `yp5`
- `xx1 ... zz1` and `xx2 ... zz2`
- `NiFrac`
- `seed`
- `disX`, `disY0`, `disZ`
- `cutOff`
- `pEnergy`
- `boundary_type`
- `component`
- `potential_alloy`
- `potential_fs`

## File handoff into the next step

The important output of Step 2 is the restart file written by:

```text
write_restart restart.al_${boundary_type}_${component}
```

That restart becomes the input to Step 3.

## What a newcomer should learn from this example

- why rigid-body translations matter,
- why consistent minimization matters,
- how GB energy is computed from a reference bulk energy,
- why this step is usually automated even though it is conceptually a single step.
